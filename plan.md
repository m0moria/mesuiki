### 1. 게임 개요
**장르 및 서브장르**  
- 장르: 로그라이크 육성 시뮬레이션 + 누키게(抜きゲー)  
- 서브장르: 타락·조교·하렘 육성 + 가챠 요소 + 선택지 분기 엔딩

**핵심 컨셉 (한 문장)**  
플레이어는 특수 정액과 신의 손가락을 가진 조교사로서, 저출산 디스토피아 세계의 미소녀들을 국가 공인 ‘S랭크 암컷’으로 육성·타락시키는 로그라이크 육성 시뮬레이터

**타겟 플레이어**  
- 연령: 19~35세 남성 (주 타겟 20대 중반~30대 초반)  
- 선호 게임 스타일: 우마무스메, 학원 아이돌 마스터, 파워프로 성공모드, 에로게(특히 조교·타락·임신물), 가챠 게임을 즐기는 층  
- 심리적 키워드: 정복욕, 소유욕, 타락 쾌감, 하렘 지배감, “내가 바꿔놓았다”는 후기감

**플레이 시간**  
- 1회 육성 루트(한 히로인 완주): 40~90분 (선택지·이벤트 스킵 여부에 따라)  
- 풀 컬렉션·모든 배드엔딩 회수 목표: 30~60시간 이상

### 2. 게임플레이 메커니즘

**핵심 게임 루프 (반복 패턴)**  
1. 턴 선택 → 훈련/이벤트/휴식/대화 중 하나 선택  
2. 훈련 실행 → 스탯 변화 + CG/대사/연출 발생  
3. 컨디션·멘탈 체크 → 과도한 훈련 시 반발·탈주·질투 이벤트 발생 가능  
4. 턴 종료 → 다음 턴로 진행 or 랜덤 이벤트 발동  
5. 주기적 평가(중간고사·기말고사) → 랭크 상승 or 강등  
6. 최종 URA 파이널 → 엔딩 분기

**주요 조작 방식**  
- 마우스/터치 중심 (PC & 모바일 웹 모두 대응 목표)  
- 클릭으로 선택지 고르기, 드래그로 스탯 그래프 확인, 더블클릭으로 스킵  
- 키보드 단축키 (1~5번으로 훈련 메뉴 빠른 선택 가능)

**게임 진행 방식**  
- 턴제 로그라이크 (고정 40~50턴 내에 결말 도달)  
- 한 번의 플레이 = 한 명의 히로인 집중 육성 (다른 히로인은 서포트 카드 형태로 등장)

**승리 조건 / 패배 조건**  
- 승리: URA 파이널 S랭크 이상 달성 + 히로인 타락도 90% 이상  
- 패배(배드엔딩): 타락도 30% 미만으로 졸업 시, 또는 특정 이벤트 실패 시 강등·추방·공중변소 엔딩

**난이도 커브**  
- 초반(1~15턴): 쉽고 친절, 실패해도 큰 페널티 없음 (학습 위주)  
- 중반(16~35턴): 질투·반항·멘탈 붕괴 이벤트 빈도 증가, 선택지 실수 시 배드엔딩 직행 가능  
- 후반(36~50턴): 고난이도 특훈, 서포트 카드 의존도 급상승, 거의 모든 선택이 중요

**보상 시스템**  
- 스탯 상승 외에 CG·대사·의상·엔딩 해금  
- SSR 서포트 카드 수집 (다른 히로인 조각)  
- “계승 포인트” → 이전 육성 데이터 일부 계승 가능 (재플레이 동기)

### 3. 기술적 명세 (Claude Code 구현 관점)

**플랫폼**  
웹 브라우저 (PC & 모바일)  
→ React + Vite + TypeScript + HTML5 Canvas (또는 Konva.js / PixiJS)

**그래픽 스타일**  
- 2.5D 라이브2D 스타일 (스탠딩 CG + Live2D-like 눈·입 움직임)  
- 메인 CG: 고해상도 일러스트 (1920×1080 기준)  
- UI: 반투명 블랙·레드·골드 계열 네온+고딕풍

**해상도 및 반응형**  
- 기본: 1920×1080 (16:9)  
- 모바일: 9:16 세로 → UI 재배치 + 터치 최적화 필수  
- 최소 지원: 1280×720

**사운드/음악 필요 여부**  
필수:  
- 배경음악 5~7곡 (메인BGM, H신 전용, 배드엔딩 전용)  
- 효과음: 신음 보이스 샘플, 젖는 소리, 채찍, 바이브 등 (라이브러리 활용 가능)  
- 히로인 보이스: 핵심 대사만 텍스트-to-speech 또는 더미 보이스

**로컬 스토리지 활용**  
- 진행도 저장 (현재 턴, 스탯, 해금 CG 목록)  
- 계승 데이터 (졸업한 히로인 일부 스탯/페티시 저장)  
- 설정 (음량, 텍스트 속도, CG 스킵 여부)

**외부 라이브러리 (추천)**  
- React, TypeScript  
- Zustand or Jotai (상태 관리)  
- Howler.js or Tone.js (사운드)  
- Konva.js or PixiJS (캔버스 CG 렌더링)  
- react-spring (애니메이션)  
- opentype.js (특수 폰트 렌더링)

### 4. 콘텐츠 구성 (MVP 기준 축소 버전)

**주요 화면**  
1. 타이틀 화면  
2. 히로인 선택 (가챠風 풀)  
3. 메인 육성 화면 (스탯 그래프 + 턴 선택 버튼)  
4. 이벤트/CG 전용 풀스크린  
5. 평가·결과 화면  
6. 갤러리·설정 화면

**UI 요소 (필수)**  
- 턴 카운터  
- 5개 스탯 육각 레이더 차트  
- 타락도 % 바  
- 컨디션/멘탈 게이지  
- 선택지 3~5개 버튼  
- CG 자동/수동 전환 토글  
- 스킵·로그 버튼

**게임 오브젝트 (MVP 기준)**  
- 히로인 스탠딩 + Live2D-like 눈/입 움직임  
- 서포트 카드 (아이콘 + 설명 텍스트)  
- 이벤트 배경 5~7종  
- 선택지 결과 텍스트 + 이펙트

**레벨/스테이지 구조 (턴 단위)**  
총 45턴 정도로 고정  
- 1~10턴: 기초 조교 (실패해도 큰 페널티 없음)  
- 11~25턴: 클럽 활동 + 질투 이벤트 본격화  
- 26~40턴: 고난이도 특훈 + 배드엔딩 분기 다수  
- 41~45턴: URA 파이널 직행 + 최종 선택지

### 5. MVP 정의 (Claude Code 구현 순서 추천)

**Phase 1 – 핵심 프로토타입 (1~2주 목표)**  
- 타이틀 → 히로인 선택 → 턴 선택 → 스탯 변화 → 간단한 텍스트 이벤트  
- 1명의 히로인 + 10턴 분량  
- 스탯 3개만 (감도, 순종, 타락도)

**Phase 2 – 기본 게임 플레이 가능 수준**  
- 3명 히로인 풀  
- 40턴 완성  
- CG 10장 + 선택지 분기 3종  
- 간단한 엔딩 2개 (S / D)

**Phase 3 – 완성도 향상**  
- Live2D-like 눈깜빡임·입움직임  
- 사운드 추가 (효과음 + BGM 3곡)  
- 서포트 카드 시스템  
- 배드엔딩 3종 추가  
- 갤러리 구현

**Phase 4 – 선택 사항 (시간 남으면)**  
- 계승 시스템  
- 질투·하렘 이벤트  
- URA 파이널 전용 연출  
- 모바일 세로 최적화

### 6. 개발 시 고려사항

**코드 구조 제안**  
```
src/
├── components/
│   ├── ui/          (Button, StatGraph, ChoicePanel 등)
│   ├── scene/       (Title, SelectGirl, Training, EventCG 등)
│   └── live2d/      (Live2D wrapper if used)
├── states/
│   ├── game.ts      (턴, 스탯, 히로인 데이터)
│   └── gallery.ts
├── data/
│   ├── heroines.ts  (히로인 마스터 데이터)
│   ├── events.ts    (턴 이벤트 테이블)
│   └── endings.ts
├── utils/
│   └── renderer.ts  (캔버스 CG 렌더링 헬퍼)
└── App.tsx
```

**성능 최적화 포인트**  
- CG는 미리 로드 + lazy loading  
- 텍스트는 한 번에 3줄 이하씩 출력 (타이핑 효과)  
- Live2D는 canvas offscreen rendering 고려  
- 상태 변경 시 불필요한 리렌더 방지 (useMemo, useCallback 적극 활용)

**잠재적 기술적 도전 과제**  
- 모바일에서 canvas + 터치 이벤트 충돌  
- 대량의 CG/음성 파일 로딩 지연  
- 복잡한 분기 관리 (턴×선택지×히로인 조합 폭발)

**확장 가능성**  
- 멀티플레이 불가능 (싱글플레이 전용)  
- 모바일 세로 대응은 비교적 용이 (UI 재배치만으로 가능)

### 7. 참고 게임 및 영감

1. **우마무스메 프리티 더비**  
   참고할 점: 턴제 육성 + 스탯 육각 차트 + 합숙 이벤트 + 대회 클라이맥스

2. **학원 아이돌 마스터**  
   참고할 점: 감정 곡선 중심 서사 + 실패 시에도 보상 있는 구조 + Live2D 활용

3. **対魔忍 시리즈 / 監獄戦艦**  
   참고할 점: 타락 단계별 CG 변화 + 배드엔딩의 강렬함 + 선택지로 인한 분기 폭

이 정도면 Claude Code가 단계별로 구현하면서도 방향성을 잃지 않을 수 있을 것입니다.
